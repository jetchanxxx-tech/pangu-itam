# ITAM 资产管理系统 — 用户使用说明书

## 系统简介

**ITAM (IT Asset Management)** 是一套 IT 资产全生命周期管理系统，支持服务器、虚拟机、网络设备等资产的统一管理，合同管理与文件版本化，系统接口注册中心，以及仪表盘监控。

### 系统地址

| 环境 | 地址 |
|------|------|
| 生产环境 | `https://itam-frontend.pages.dev` |
| API 地址 | `https://itam-api.<子域名>.workers.dev` |

---

## 一、登录系统

### 1.1 访问与登录

打开系统地址，进入登录页面。输入用户名和密码后点击「Sign In」。

### 1.2 默认账户

| 角色 | 用户名 | 密码 | 权限 |
|------|--------|------|------|
| 管理员 | admin | admin123 | 全部操作权限 |
| 普通用户 | user | user123 | 查看权限，部分操作受限 |

### 1.3 退出登录

点击右上角用户头像 → 选择「Logout」。

### 1.4 修改密码

首次登录后建议修改密码。当前版本通过 API 调用：
```
POST /api/v1/user/change-password
Body: { "oldPassword": "...", "newPassword": "..." }
```

## 二、仪表盘 (Dashboard)

### 2.1 概览

仪表盘展示系统的核心运营指标：

| 指标 | 说明 |
|------|------|
| 资产总数 | 系统中所有资产的数量 |
| UEBA 风险评分 | 用户行为分析评分（当前固定 15/100） |
| 活跃告警 | 离线/维护状态的资产数量 |
| SLA 合规率 | 在线资产占比，目标 > 99% |
| 待处理审计 | 等待审计的项目数（当前固定 5） |

### 2.2 系统健康

展示网络、计算、存储各组件的健康评估分数。

### 2.3 近期活动

显示最近的操作记录（登录、新增资产、告警等）。

## 三、资产管理 (Assets)

### 3.1 资产列表

左侧菜单点击「资产」进入。表格展示所有资产，支持：

- **搜索**：按名称、IP、负责人搜索
- **筛选**：按类型过滤（Server / VM / Container / Software）
- **排序**：点击表头排序
- **分页**：底部翻页，可调整每页显示数量

### 3.2 资产类型与状态

| 类型 | 说明 | 状态选项 |
|------|------|----------|
| Server | 物理服务器 | Online |
| VM | 虚拟机 | Offline |
| Container | 容器实例 | Maintenance |
| Software | 软件资产 | |

### 3.3 新增资产

1. 点击右上角「Add Asset」按钮
2. 填写资产信息：
   - **名称**（必填）：资产唯一标识
   - **类型**（必填）：Server / VM / Container / Software
   - **平台**：AWS / VMware / BareMetal / Azure 等
   - **IP 地址**（必填）：管理 IP 或端点地址
   - **状态**：Online / Offline / Maintenance
   - **负责人**：资产管理员
   - **描述**：补充说明
   - **规格**：例如 "4vCPU/16GB/500GB SSD"
3. 点击「Create」提交

### 3.4 编辑与删除

- **编辑**：点击行右侧编辑图标 → 修改表单 → 点击「Update」
- **删除**：点击删除图标 → 确认弹窗 → 确认删除

### 3.5 连接资产

点击「Connect」按钮，可选择：
- **SSH 终端**：跳转到 Web 终端页面
- **RDP 连接**：远程桌面（仅 VM/Server 类型）
- **浏览器打开**：直接在浏览器打开资产 IP

## 四、合同管理 (Contracts)

### 4.1 合同列表

展示所有合同，包含名称、编号、类型、状态、供应商、金额、有效期等。

**合同类型**：
- **采购 (procurement)**：设备或服务采购合同
- **维保 (maintenance)**：维护保养合同
- **租赁 (lease)**：设备租赁合同

**合同状态**：
- **Draft**：草稿
- **Active**：生效中
- **Expired**：已过期
- **Terminated**：已终止

### 4.2 新增合同

1. 点击「Add Contract」
2. 填写合同信息（名称必填，其他可选）：
   - 名称、编号、类型、状态
   - 供应商、金额、币种（CNY/USD）
   - 起止日期、签署日期
   - 负责人、联系方式、描述
   - 关联资产 ID（可选）
3. 点击「Create」

### 4.3 合同文件管理

点击文件图标进入文件管理弹窗。

#### 上传文件
1. 点击「Upload File」
2. 选择文件（单个文件 ≤ 10MB）
3. 自动版本递增（v1, v2, v3...）

#### 下载文件
点击文件右侧「Download」按钮。

#### 删除文件
点击红色删除按钮 → 确认。

**支持的文件类型**：PDF、Word、Excel、图片等所有常见格式。

## 八、数据导入 (Import)

### 8.1 导入流程

1. 下载模板（CSV 格式，含 BOM 头适配 Excel）
2. 按模板格式填写数据
3. 拖拽或选择文件上传
4. 系统解析并导入

### 8.2 CSV 模板格式

```csv
名称,类型,平台,IP地址,状态,负责人
Server-Prod-01,Server,AWS,10.0.1.100,Online,张三
VM-Dev-02,VM,VMware,192.168.1.50,Offline,李四
```

## 十一、常见问题 (FAQ)

### Q: 忘记密码怎么办？
A: 联系管理员在后台重置密码，或通过 API 的 `/api/v1/user/change-password` 接口修改。

### Q: 合同文件上传失败？
A: 检查文件大小是否超过 10MB 限制，以及文件格式是否被支持。如超过 10MB，请压缩后上传。

### Q: 资产列表加载慢？
A: 系统使用分页加载，默认每页 20 条。可以缩小每页数量或使用搜索筛选举。

### Q: 如何备份数据？
A: 使用 `npx wrangler d1 export itam-db` 导出 D1 数据库快照。

### Q: 如何添加新用户？
A: 通过 D1 数据库直接插入用户记录，密码使用 PBKDF2 SHA-256 哈希存储。
